  <behavior>
    SLOT-01 behaviors:
    - findCandidates queries "appointments" table (not waitlist_entries) for scheduled/confirmed/reminder_pending/reminder_sent appointments
    - Only includes appointments with scheduled_at AFTER the open slot time
    - Excludes the cancelling patient (by cancellingPatientId in OpenSlotDetails)
    - Excludes patients who declined any offer in the last 24 hours (query waitlist_offers for declined + responded_at > 24h ago)
    - Excludes appointments that time-conflict with the open slot (range overlap: candidateStart < slotEnd AND slotStart < candidateEnd)
    - Deduplicates by patient_id, keeping the farthest-out appointment (maximizes "moving earlier" value)
    - Returns empty array when slot is in the past (scheduledAt <= now)
    - Returns empty array when lead time is less than 2 hours (scheduledAt - now < 2 hours)
    - Caps results at `limit` parameter (default 50)

    SLOT-02 behaviors:
    - Each candidate is scored via computeCandidateScore (appointmentDistance + reliability)
    - No-show stats are fetched from appointments table grouped by patient_id
    - Candidates are sorted by candidateScore.total descending
    - Result uses new RankedCandidate interface with candidateAppointmentId (not waitlistEntryId)

    Edge cases:
    - Patient with 3 future appointments: appears once with farthest appointment
    - Zero candidates after all filters: returns empty array
    - Database error on candidate query: logs error, returns empty array
  </behavior>

     a. Guard: if slot.scheduledAt <= now, return []. If slot.scheduledAt - now < 2 hours, return [].
     b. Query appointments table:
        - `.from("appointments").select("*, patient:patients(id, first_name, last_name, phone, email, preferred_channel)")`
        - `.eq("tenant_id", slot.tenantId)`
        - `.in("status", ["scheduled", "reminder_pending", "reminder_sent", "confirmed"])`

     i. Sort by `candidateScore.total` descending.

     function hasTimeConflict(
       candidateStart: Date, candidateDurationMin: number,
       slotStart: Date, slotDurationMin: number

       const candidateEnd = new Date(candidateStart.getTime() + candidateDurationMin * 60_000);
       const slotEnd = new Date(slotStart.getTime() + slotDurationMin * 60_000);
       return candidateStart < slotEnd && slotStart < candidateEnd;
     }
