<objective>
Add channel-level reconnection with exponential backoff, stale data recovery on reconnect, and background tab resilience to the existing useRealtimeAppointments hook.

Purpose: The Supabase Realtime transport layer handles brief network blips automatically, but channel-level failures (CLOSED, TIMED_OUT, CHANNEL_ERROR) require application code to tear down the dead channel and re-subscribe. Without this, the dashboard silently goes stale after any extended network disruption.

Output: Enhanced Supabase client with `worker: true` for background tab heartbeat survival, enhanced hook with `reconnectTrigger` pattern for channel-level reconnection, and stale data recovery via REST re-fetch on successful re-subscription.
</objective>

Enhance the existing `useRealtimeAppointments` hook with three reconnection mechanisms. The approach uses the `reconnectTrigger` pattern (Research Pattern 3): a state counter that forces the useEffect to re-run, which naturally tears down the old channel via cleanup and creates a fresh one.

**New state and refs to add (at the top of the hook, alongside existing state):**

1. `const [reconnectTrigger, setReconnectTrigger] = useState(0);` -- Forces useEffect re-run when incremented
2. `const reconnectTimeoutRef = useRef<ReturnType<typeof setTimeout> | null>(null);` -- Stores pending reconnection timer to prevent accumulation (Pitfall 4)
3. `const reconnectAttemptRef = useRef(0);` -- Tracks consecutive failed attempts for exponential backoff
4. `const hasBeenSubscribedRef = useRef(false);` -- Distinguishes first connection from re-connection for stale data recovery
5. `const currentStatusRef = useRef<RealtimeStatus>("CONNECTING");` -- Tracks current status for visibilitychange handler

**Import additions:** Add `RECONNECTABLE_STATUSES` and `MAX_RECONNECT_ATTEMPTS` from `@/lib/realtime/types`.

**Modify the `.subscribe()` callback (replacing the existing one-liner):**

```typescript
.subscribe((status) => {
  const mappedStatus = status as RealtimeStatus;
  setRealtimeStatus(mappedStatus);
  currentStatusRef.current = mappedStatus;

  if (mappedStatus === "SUBSCRIBED") {
    // Reset reconnect counter on success
    reconnectAttemptRef.current = 0;

    // Stale data recovery: if this is a RE-subscription, re-fetch all data
    if (hasBeenSubscribedRef.current) {
      fetchInitial();
    }
    hasBeenSubscribedRef.current = true;
  }

  if (RECONNECTABLE_STATUSES.includes(mappedStatus)) {
    // Channel-level reconnection with exponential backoff
    if (reconnectAttemptRef.current < MAX_RECONNECT_ATTEMPTS) {
      // Clear any existing timer to prevent accumulation (Pitfall 4)
      if (reconnectTimeoutRef.current !== null) {
        clearTimeout(reconnectTimeoutRef.current);
      }
      const delay = Math.min(
        1000 * Math.pow(2, reconnectAttemptRef.current),
        30_000,
      );
      reconnectAttemptRef.current += 1;
      reconnectTimeoutRef.current = setTimeout(() => {
        reconnectTimeoutRef.current = null;
        setReconnectTrigger((prev) => prev + 1);
      }, delay);
    }
  }
})
```

**Add visibilitychange listener inside the useEffect (after fetchInitial call, before return):**

```typescript
const handleVisibilityChange = () => {
  if (
    document.visibilityState === "visible" &&
    currentStatusRef.current !== "SUBSCRIBED"
  ) {
    // Tab became visible and connection is not live -- reset counter and reconnect
    reconnectAttemptRef.current = 0;
    setReconnectTrigger((prev) => prev + 1);
  }
};
document.addEventListener("visibilitychange", handleVisibilityChange);
```

This handles the "overnight" case: after max attempts are exhausted and the indicator shows "Offline", returning to the tab resets the counter and tries again.
