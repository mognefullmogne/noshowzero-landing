<objective>
Fix inflated revenue metrics to count only real slot recoveries and add per-tenant configurable appointment value.

Purpose: The current analytics endpoint inflates "revenue recovered" by counting ALL confirmed/completed appointments multiplied by a hardcoded value. This phase fixes it to count only appointments that actually filled a cancelled/no-show slot via the cascade, uses a per-tenant configurable appointment value, and corrects the fill rate formula per METR-04.

Output: Migration for tenant appointment value, pure metric functions with tests, fixed analytics API, tenant settings API endpoint.
</objective>
