1. Modify `src/hooks/use-realtime-appointments.ts` to add toast notifications:
   - Add import: `import { toast } from "sonner";`
   - In the `postgres_changes` callback, AFTER the delta-merge (`setAppointments(prev => applyDelta(prev, payload))`), add toast logic for confirmation events:
     ```typescript
     // Fire toast when an appointment is confirmed
     if (payload.eventType === 'UPDATE') {
       const newRow = payload.new as Record<string, unknown>;
       const oldRow = payload.old as Record<string, unknown>;
       if (newRow.status === 'confirmed' && oldRow.status !== 'confirmed') {
         toast.success('Appuntamento confermato', {
           description: (newRow.service_name as string) ?? 'Appuntamento',
           duration: 4500,
         });
       }
     }
     ```
   - Also add the toast logic in the pending events drain loop (for events that arrived during initial fetch). When draining, apply the same confirmation check after each `applyDelta` call.
   - IMPORTANT: The toast should only fire for events received AFTER the initial load, not for the initial fetch data. The pending events queue may contain events from the gap between subscribe and fetch -- those are valid toast candidates. But the initial fetch data itself should NOT trigger toasts. The current architecture handles this correctly because toasts are fired in the `on('postgres_changes', ...)` callback and the pending drain, not during `fetchInitial`'s `setAppointments(data)`.
