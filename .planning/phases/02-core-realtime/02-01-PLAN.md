Create `src/lib/realtime/apply-delta.ts` with:
- Export `applyDelta(current: readonly Appointment[], payload: RealtimeAppointmentEvent): readonly Appointment[]`
- IMMUTABLE -- always return new arrays, never mutate.
- INSERT: Cast `payload.new` as `Appointment`. Check for duplicates by id (idempotent). Prepend to array.
- UPDATE: Cast `payload.new` as partial Appointment. Find existing by id. If found, merge with spread BUT preserve the existing `patient` field (Realtime events do NOT include JOINed data -- see Pitfall 1 in RESEARCH.md). If not found, append (it may be a new appointment entering the user's filter view).
- DELETE: Cast `payload.old` as `{ id: string }`. Filter out by id.
- Default case: return `current` unchanged.
- This function must be pure -- no side effects, no imports beyond types.

<task type="auto">
  <name>Task 2: Create useRealtimeAppointments hook</name>
  <files>src/hooks/use-realtime-appointments.ts</files>
  <action>
Create `src/hooks/use-realtime-appointments.ts` exporting `useRealtimeAppointments(tenantId: string | undefined)`.

Implementation follows the subscribe-first-then-fetch pattern from RESEARCH.md Pattern 1:

1. State: `appointments` (readonly Appointment[]), `loading` (boolean), `realtimeStatus` (string).
2. Refs: `pendingEventsRef` (queues events before initial fetch completes), `initializedRef` (boolean flag).
3. `useEffect` keyed on `[tenantId]`:
   - Early return if `!tenantId`.
   - Reset refs: `initializedRef.current = false`, `pendingEventsRef.current = []`.
   - Create Supabase client via `createClient()` from `@/lib/supabase/client`.
   - SUBSCRIBE FIRST: `supabase.channel(\`appointments:\${tenantId}\`)` -- use tenant-specific channel name per RESEARCH Pattern 4.
     - `.on('postgres_changes', { event: '*', schema: 'public', table: 'appointments' }, callback)` -- do NOT add `filter` for tenant_id (RLS handles tenant scoping server-side per SEC-01, see RESEARCH Anti-Patterns).
     - In the callback: if `!initializedRef.current`, push to `pendingEventsRef.current` and return. Otherwise, call `setAppointments(prev => applyDelta(prev, payload))`.
     - `.subscribe((status) => setRealtimeStatus(status))` -- track subscription status.
   - THEN FETCH: async function `fetchInitial()`:
     - Query `supabase.from('appointments').select('*, patient:patients(*)').eq('tenant_id', tenantId).order('scheduled_at', { ascending: false })`.
     - Set appointments state, set loading false.
     - Set `initializedRef.current = true`.
     - Drain `pendingEventsRef.current` by applying each queued event via `applyDelta`.
     - Clear the queue.
   - Call `fetchInitial()`.
   - CLEANUP: return `() => { supabase.removeChannel(channel); }` -- use `removeChannel` not `unsubscribe` per RESEARCH.

4. Return `{ appointments, loading, realtimeStatus }` matching `UseRealtimeAppointmentsReturn`.

Import conventions: Use `@/` absolute paths. Put external imports first, then `@/lib`, then `@/hooks` (but this hook should NOT import from `@/hooks/use-tenant` -- the caller provides `tenantId`).

Do NOT add toast logic here -- that belongs in Plan 03 (separation of concerns).
